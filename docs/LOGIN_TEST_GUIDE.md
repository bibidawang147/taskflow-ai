# 登录系统测试指南

## ✅ 登录系统已完成

登录和注册功能已经完全集成后端 API，现在可以正常使用！

## 🚀 快速测试

### 1. 启动服务

**启动后端：**
```bash
cd backend
npm run dev
```
后端运行在 http://localhost:5000

**启动前端：**
```bash
cd frontend
npm run dev
```
前端运行在 http://localhost:3000

### 2. 注册新账号

1. 访问 http://localhost:3000/register
2. 填写信息：
   - 用户名：随意填写（如：测试用户）
   - 邮箱：随意填写（如：test@example.com）
   - 密码：至少6位
   - 确认密码：与密码相同

3. 点击"创建账户"
4. 注册成功后会显示：
   - ✅ "注册成功！已赠送 50,000 积分，即将跳转..."
   - 自动跳转到首页
   - 导航栏显示余额和用户菜单

### 3. 登录账号

1. 访问 http://localhost:3000/login
2. 输入刚才注册的邮箱和密码
3. 点击"登录"
4. 登录成功后：
   - 自动跳转到首页
   - 导航栏显示余额（50,000 coins）
   - 可以看到"AI 对话"、"统计"、"会员"等菜单

### 4. 测试功能

**查看余额：**
- 导航栏右上角显示余额按钮
- 点击可跳转到充值页面

**AI 对话：**
1. 点击导航栏"AI 对话"
2. 选择模型（如 GPT-4o Mini）
3. 输入消息测试
4. 查看返回结果和消耗

**使用统计：**
- 访问 `/usage-stats`
- 查看详细的使用记录

**会员中心：**
- 访问 `/membership`
- 可以升级会员（演示环境免费）

**充值中心：**
- 访问 `/recharge`
- 查看充值套餐

### 5. 退出登录

1. 点击导航栏右上角的用户图标
2. 点击"退出登录"
3. 自动跳转到登录页

## 🔧 功能说明

### 注册功能

**✅ 已实现：**
- 真实的后端 API 调用
- 密码强度检测
- 表单验证
- 错误提示
- 成功提示
- 自动登录和跳转
- **自动初始化余额（赠送 50,000 积分）**

**注册时会：**
1. 创建用户账号
2. 自动初始化积分余额
3. 赠送 50,000 积分
4. 设置每日免费额度（10,000 coins）
5. 保存 JWT Token
6. 自动跳转到首页

### 登录功能

**✅ 已实现：**
- 真实的后端 API 调用
- 表单验证
- 错误处理
- JWT Token 保存
- 自动跳转
- 导航栏状态更新

**登录时会：**
1. 验证邮箱和密码
2. 获取 JWT Token
3. 保存到 localStorage
4. 刷新页面更新状态
5. 显示用户余额

### Token 管理

**Token 保存位置：**
- localStorage（key: 'token'）

**Token 使用：**
- 所有 API 请求自动添加 `Authorization: Bearer <token>` 头
- API 拦截器自动处理

**Token 失效：**
- 401 错误自动清除 token
- 自动跳转到登录页

## 🐛 常见问题

### Q1: 注册后没有跳转？

**检查：**
1. 浏览器控制台是否有错误
2. 后端是否正常运行
3. 网络请求是否成功（查看 Network 标签）

**解决：**
```bash
# 重启后端
cd backend
npm run dev

# 重启前端
cd frontend
npm run dev
```

### Q2: 登录显示"邮箱或密码错误"？

**确认：**
1. 使用的是注册时的邮箱和密码
2. 密码区分大小写
3. 邮箱格式正确

### Q3: 导航栏没有显示余额？

**检查：**
1. 是否已登录（localStorage 中是否有 token）
2. 刷新页面
3. 查看浏览器控制台是否有错误

**解决：**
```javascript
// 在浏览器控制台执行
localStorage.getItem('token')  // 查看 token 是否存在

// 如果没有 token，重新登录
```

### Q4: API 请求失败？

**检查后端：**
```bash
cd backend

# 确保数据库已初始化
npx prisma db push
npx ts-node src/scripts/seed-model-pricing.ts

# 启动服务
npm run dev
```

**检查前端环境变量：**
```bash
# frontend/.env
VITE_API_URL=http://localhost:5000/api
```

### Q5: 余额为 0？

**原因：**
- 老用户未初始化余额

**解决：**
```bash
# 方法1：重新注册一个账号

# 方法2：后端手动初始化
# 在后端项目根目录创建脚本 fix-balance.ts
```
