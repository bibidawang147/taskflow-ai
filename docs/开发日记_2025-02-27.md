# 开发日记 2025-02-27

## 一、画布系统优化（StoragePage.tsx）

### 1. 修复新建容器按钮无反应
- **问题**：点击"新建容器"后画布上看不到新容器。根本原因是 `findNonOverlappingPosition` 螺旋搜索在已有大量元素时，将新容器放置到了天文坐标（如 x:69837, y:69862），超出可视范围。
- **修复**：重写 `handleCreateContainer`，新容器始终放在现有内容最底部 +20px 间距处，不再使用螺旋搜索。
- **清理**：删除了数据库中 62 个历史遗留的空白容器。

### 2. 容器 UI 改版
- 边框改为**紫色虚线**（`2px dashed rgba(139, 92, 246, 0.4)`）
- 背景改为**透明**（原来是白色 `#ffffff`）

### 3. 连接线体验优化
- **问题**：连接线有时连不上目标手柄，因为手柄仅 7px 大小，且使用 `elementFromPoint` 单像素检测。
- **修复**：
  - 增加 20px 屏幕距离的兜底检测策略（遍历所有手柄找最近的）
  - 连接模式下手柄放大 1.6 倍 + 紫色发光效果
  - 拖拽连接线时 30px 范围内**磁吸吸附**到最近手柄，线条自动变实线加粗

### 4. 修复连接线起点偏移（容器内卡片）
- **问题**：容器内的卡片，连接线起点/终点偏移约 16px，不在卡片手柄上。
- **原因**：`getContainerContentOrigin` 在计算内容原点时额外加了 `CONTAINER_PADDING`（16px），但 DOM 中卡片使用 `position: absolute`，CSS padding 不会偏移绝对定位元素，导致 SVG 坐标与 DOM 渲染位置不一致。
- **修复**：从 `getContainerContentOrigin` 中移除 `CONTAINER_PADDING` 偏移，使模型坐标与 DOM 渲染一致。

### 5. 标签页懒加载
- 新增 `mountedTabIds` 状态，只有被访问过的 tab 才会挂载组件
- 避免刷新时所有 tab 同时加载导致多个失败 toast

### 6. 侧边栏删除工作流与标签页联动
- 删除已在标签页打开的工作流时，显示三按钮弹窗：取消 / 仅删除（保留标签） / 删除并关闭标签
- 被删除的标签显示红色上边框 + "已删除"标记

## 二、工作流编辑器增强（WorkflowCreatePage.tsx）

### 1. 预期结果支持附件
- 每个步骤的"预期结果"区域新增图片/文件上传功能
- 支持**直接粘贴图片**（onPaste 事件检测 clipboard 中的图片）
- 附件通过 `/api/utils/upload-media` 或 `/api/utils/upload-document` 上传
- 新增 `expectedResultAttachments` 字段，保存并同步到后端

### 2. 文章转工作流 Prompt 大幅优化
- 原来的 prompt 过于简单（只要求标题/描述/步骤）
- 新 prompt 定义了完整的角色、目标、字段规范（tools/prompts/media/documents四类资源）
- 增加智能拆分规则（粒度判断、顺序保持、冗余合并、隐含步骤补全）
- 文章输入方式从"URL或内容"改为"纯粘贴内容"

### 3. 自动恢复已删除工作流
- autoSave 和手动保存时，如果后端返回 404（工作流已被删除），自动创建新工作流
- 通过 `onWorkflowIdChange` 通知父组件更新标签页的 workflowId

### 4. 嵌入式编辑 URL 隔离
- 在工作台标签页内编辑工作流时，不再调用 `window.history.replaceState` 修改浏览器 URL
- 避免刷新页面后跳离画布进入独立编辑页
