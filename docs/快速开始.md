## 💡 核心概念解释

### 1. 什么是 SQLite？

**SQLite = 一个文件就是一个数据库**

```
你可能听说过的数据库：
- MySQL: 需要安装服务器 ❌
- PostgreSQL: 需要安装服务器 ❌
- MongoDB: 需要安装服务器 ❌

SQLite:
- 就是一个文件（dev.db）✅
- 不需要安装任何东西 ✅
- 放在项目里，随时可用 ✅
```

**位置**: `/Users/star/Desktop/开发文件夹/workflow-platform/backend/prisma/dev.db`

### 2. 开发环境 vs 生产环境

```
开发环境（现在）：
📍 在你的电脑上
💾 SQLite 数据库（一个文件）
🔧 用于学习和测试
👤 只有你能访问

生产环境（部署后）：
🌐 在云服务器上
💾 PostgreSQL/MySQL（需要真实数据库服务器）
🚀 用户可以访问
👥 对外提供服务
```

**结论**: 现在不用担心生产环境，先在本地开发！

## 🎯 立即开始的 3 个步骤

1. 打开浏览器，访问 **http://localhost:5555**
2. 你会看到所有的数据表
3. 点击 `users` 表，查看用户数据（现在是空的）
4. 试着手动添加一个用户：

### Step 2: 测试后端 API（5 分钟）

#### 方法 1: 使用 curl（命令行）

打开终端，运行：

```bash
# 1. 测试健康检查
curl http://localhost:8000/health

# 2. 注册一个用户
curl -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "123456",
    "name": "测试用户"
  }'

# 3. 登录
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "123456"
  }'
```

运行完后，再打开 **http://localhost:5555**，你会在 `users` 表里看到刚注册的用户！

#### 方法 2: 使用 Postman（图形界面，推荐）

1. 下载安装 Postman: https://www.postman.com/downloads/
2. 打开 `test-api.http` 文件，里面有所有测试用例
3. 在 Postman 中一个个测试

## 📖 数据库操作速查

### 如何添加新的数据表？

编辑 `backend/prisma/schema.prisma`:

```prisma
// 1. 添加新模型
model WorkItem {
  id          String   @id @default(cuid())
  name        String
  icon        String
  category    String
  createdAt   DateTime @default(now())

  @@map("work_items")
}
```

然后运行：

```bash
cd backend
npx prisma db push
```

刷新 http://localhost:5555，你会看到新表！

### 如何查询数据？

在后端代码中：

```typescript
// 获取所有用户
const users = await prisma.user.findMany()

// 获取单个用户
const user = await prisma.user.findUnique({
  where: { id: userId }
})

// 创建用户
const newUser = await prisma.user.create({
  data: {
    email: 'test@example.com',
    name: '测试',
    password: 'hashed-password'
  }
})
```

## ❓ 常见问题

### Q: 我的数据会丢吗？
**A**: 不会！SQLite 保存在 `dev.db` 文件中，只要文件在，数据就在。

### Q: 多人协作怎么办？
**A**:
- 开发环境：每个人有自己的 `dev.db`
- 生产环境：所有人连接同一个云数据库

### Q: 我需要安装 MySQL 或 PostgreSQL 吗？
**A**: **不需要！** 开发时用 SQLite 就够了。

### Q: SQLite 性能够吗？
**A**:
- 开发/学习：完全够用 ✅
- 小型应用（<10万用户）：够用 ✅
- 大型应用：需要 PostgreSQL/MySQL

### Q: 什么时候需要切换到 PostgreSQL？
**A**:
- 部署到生产环境时
- 需要多服务器同时访问时
- 用户量很大时

但现在不用管这些！
