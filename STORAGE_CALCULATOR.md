# 存储空间计算器

## 📦 单条笔记数据大小（实测）

### 不下载图片/视频（当前配置）
```
- 标题 + 描述：~1KB
- 3-5条评论：~1-2KB
- 元数据（点赞、收藏等）：~0.5KB
- 总计：~3-5KB/条
```

### 如果下载图片（不推荐）
```
- 单条笔记：~500KB-2MB
- 不建议开启
```

## 📈 批量爬取预估

### 小规模（初始测试）
```
50 条笔记
├─ JSON数据：50 × 5KB = 250KB
├─ 数据库：~50KB
└─ 总计：<1MB ✅
```

### 中等规模（建立基础库）
```
200 条笔记
├─ JSON数据：200 × 5KB = 1MB
├─ 数据库：~200KB
└─ 总计：~1.5MB ✅
```

### 大规模（完整工作流库）
```
1000 条笔记
├─ JSON数据：1000 × 5KB = 5MB
├─ 数据库：~1MB
└─ 总计：~6MB ✅
```

### 超大规模（持续积累）
```
5000 条笔记
├─ JSON数据：5000 × 5KB = 25MB
├─ 数据库：~5MB
├─ 浏览器缓存：~200MB（一次性）
└─ 总计：~230MB ✅
```

## 🎯 你的 6.7GB 空间可以：

### 保守估计（只算爬虫相关）
```
6.7GB 可用空间
- 浏览器缓存：-200MB（一次性）
- 系统预留：-500MB（安全边际）
= 6GB 可用于数据

6GB ÷ 5KB = 1,200,000 条笔记
```

**结论：即使爬 10万条也只占 500MB！**

## 💡 实际推荐配置

### 方案A：快速建库（推荐）
```python
CRAWLER_MAX_NOTES_COUNT = 20  # 每个关键词 20 条
maxCount = 20                 # API 调用时设置

预估：
- 10个关键词 × 20条 = 200条
- 占用空间：~1MB
- 时间：~30分钟
```

### 方案B：大规模采集
```python
CRAWLER_MAX_NOTES_COUNT = 50  # 每个关键词 50 条
maxCount = 50

预估：
- 20个关键词 × 50条 = 1000条
- 占用空间：~5MB
- 时间：~2小时
```

### 方案C：持续采集
```python
# 定时任务，每天爬取
每天 5 个关键词 × 10 条 = 50 条/天
一个月 = 1500 条
占用空间：~7.5MB/月
```
