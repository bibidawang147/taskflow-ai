# 部署指南

## 🚀 快速部署（推荐）

### 修改代码后的完整流程

```bash
# 1. 修改代码（在 VSCode 或任何编辑器中）
# 2. 在项目根目录运行：

./deploy.sh
```

就这么简单！脚本会自动：
- ✅ 构建前端和后端代码
- ✅ 上传到服务器
- ✅ 安装依赖（如果有变化）
- ✅ 重启服务
- ✅ 显示运行状态

---

## 📋 常用命令

### 只部署前端
```bash
./deploy.sh frontend
# 或
./deploy.sh fe
```

### 只部署后端
```bash
./deploy.sh backend
# 或
./deploy.sh be
```

### 部署全部（前后端）
```bash
./deploy.sh all
# 或
./deploy.sh
```

### 快速重启（不重新构建）
```bash
./deploy.sh restart
```
适用场景：只修改了 .env 环境变量

### 查看服务状态
```bash
./deploy.sh status
```

## 🔄 完整的开发到部署流程

### 日常开发流程

```bash
# 1️⃣ 本地开发
cd frontend  # 或 backend
npm run dev  # 启动开发服务器

# 2️⃣ 修改代码

# 3️⃣ 本地测试
# 确保功能正常

# 4️⃣ 部署到生产
cd ..  # 回到项目根目录
./deploy.sh  # 一键部署

# 5️⃣ 验证部署
# 浏览器访问: http://47.93.218.80
```

## 🎯 不同场景的部署策略

| 修改内容 | 需要部署的部分 | 命令 |
|---------|--------------|------|
| 只改前端页面/样式 | 前端 | `./deploy.sh frontend` |
| 只改后端 API | 后端 | `./deploy.sh backend` |
| 修改数据库 schema | 后端 + 数据库迁移 | 见下方 |
| 修改 .env 配置 | 重启服务 | `./deploy.sh restart` |
| 新增 npm 依赖 | 完整部署 | `./deploy.sh` |
| 前后端都改了 | 全部 | `./deploy.sh all` |

## 🗄️ 数据库迁移

如果修改了 `prisma/schema.prisma`：

```bash
# 1. 本地测试迁移
cd backend
npx prisma migrate dev

# 2. 部署到服务器
./deploy.sh backend

# 3. 在服务器上运行迁移
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80 << 'EOF'
  cd /root/workflow-backend
  npx prisma migrate deploy
  pm2 restart workflow-backend
EOF
```

## ⚡ 性能优化建议

### 减少部署时间

1. **只部署改动的部分**
   - 前端改动 → `./deploy.sh frontend`
   - 后端改动 → `./deploy.sh backend`

2. **使用快速重启**
   - 只改配置 → `./deploy.sh restart`

3. **本地先测试**
   - 确保 `npm run build` 成功再部署
