## 🎯 任务一：前端部署（今天完成）

### 预计时间：2-3小时

### 📖 参考文档
`frontend/前端部署指南.md`

### 🔑 关键步骤

#### Step 1: 准备工作 (15分钟)

```bash
# 进入前端目录
cd /Users/star/Desktop/开发文件夹/workflow-platform/frontend

# 创建生产环境配置
cat > .env.production << 'EOF'
VITE_API_BASE_URL=http://47.93.218.80:3000
VITE_APP_ENV=production
EOF

# 检查 src/services/api.ts 配置
# 确保使用 import.meta.env.VITE_API_BASE_URL
```

#### Step 2: 构建前端 (5分钟)

```bash
npm run build

# 验证构建产物
ls -lh dist/
```

#### Step 3: 上传到服务器 (5分钟)

```bash
# 方式1：上传压缩包
cd dist
tar -czf frontend-dist.tar.gz *
scp -i /Users/star/Desktop/工作流网站.pem frontend-dist.tar.gz root@47.93.218.80:/root/

# 方式2：直接上传目录
scp -i /Users/star/Desktop/工作流网站.pem -r dist root@47.93.218.80:/root/workflow-frontend
```

#### Step 4: 服务器配置 (60分钟)

**SSH 登录服务器：**

```bash
ssh -i /Users/star/Desktop/工作流网站.pem root@47.93.218.80
```

**安装 Nginx：**

```bash
yum install -y nginx
systemctl start nginx
systemctl enable nginx
```

**解压前端文件：**

```bash
mkdir -p /var/www/workflow-platform
cd /var/www/workflow-platform

# 如果是压缩包
tar -xzf /root/frontend-dist.tar.gz

# 如果是目录
mv /root/workflow-frontend/* .
```

**配置 Nginx：**

```bash
cat > /etc/nginx/conf.d/workflow.conf << 'EOF'
server {
    listen 80;
    server_name 47.93.218.80;

    root /var/www/workflow-platform;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
EOF

# 测试配置
nginx -t

# 重启
systemctl restart nginx
```

**开放端口：**

```bash
# 防火墙
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --reload

# 阿里云安全组（浏览器操作）
# https://ecs.console.aliyun.com
# 添加入站规则：80/80，0.0.0.0/0
```

#### Step 5: 测试 (10分钟)

**浏览器访问：**

```
http://47.93.218.80
```

**检查项：**
- [ ] 首页能打开
- [ ] 样式正常显示
- [ ] 图片能加载
- [ ] API 调用正常（查看 Network 标签）
- [ ] 登录功能正常

**如果有问题：**

```bash

tail -f /var/log/nginx/error.log

# 查看后端日志

# 检查后端是否运行
```

## 🆘 遇到问题？

### 前端无法访问

1. 检查 Nginx 状态：`systemctl status nginx`
2. 查看错误日志：`tail -f /var/log/nginx/error.log`
3. 检查防火墙：`firewall-cmd --list-ports`
4. 检查安全组：阿里云控制台

### API 调用失败

1. 检查后端运行：`pm2 list`
2. 查看后端日志：`pm2 logs workflow-backend`
3. 检查 CORS 配置：后端 server.ts
4. 测试 API：`curl http://localhost:3000/health`

### Sentry 未收到错误

1. 检查 DSN 配置：`.env` 文件
2. 检查初始化代码：server.ts / main.tsx
3. 触发测试错误验证
4. 查看控制台输出

### 测试运行失败

1. 检查依赖安装：`npm install`
2. 检查配置文件：jest.config.js / vitest.config.ts
3. 查看错误信息
4. 参考指南逐步检查
