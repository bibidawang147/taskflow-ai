# 拖拽避让 - 丝滑度优化指南

## 🎯 优化目标

解决元素"粘连"问题，实现真正独立、丝滑的避让效果。

## 🔧 核心优化

### 问题分析
**优化前的问题**：
- ❌ 元素避让后"粘"在新位置
- ❌ 拖拽元素略过后，其他元素不会恢复原位
- ❌ 元素之间缺乏独立性

**根本原因**：
- 只计算了避让，没有计算恢复
- 没有记录元素的原始位置
- 拖拽元素离开后，被避让的元素不知道回到哪里

### 解决方案

#### 1. 原始位置记录系统

```typescript
// 在拖拽开始时保存所有元素的原始位置
const originalPositionsRef = useRef<Record<string, Position>>({})

// handleItemMouseDown 中
if (isAvoidanceEnabled) {
  const originalPositions = {}
  Object.entries(canvasItems).forEach(([id, item]) => {
    originalPositions[id] = { ...item.position }
  })
  originalPositionsRef.current = originalPositions
}
```

**作用**：记住每个元素拖拽开始时的位置，作为恢复的目标

#### 2. 双态避让算法

```typescript
// 返回值可以是新位置或 'restore' 标记
type AvoidanceResult = Record<string, Position | 'restore'>

// 算法同时处理：
// 1. 正在碰撞的元素 -> 计算避让位置
// 2. 不再碰撞的元素 -> 返回 'restore' 标记
```

**作用**：区分避让和恢复两种状态

#### 3. 智能恢复逻辑

```typescript
for (const [itemId, newPos] of Object.entries(avoidancePositions)) {
  if (newPos === 'restore') {
    // 恢复到原始位置
    const originalPos = originalPositionsRef.current[itemId]
    if (originalPos) {
      draft[itemId] = { ...item, position: originalPos }
    }
  } else {
    // 应用避让位置
    draft[itemId] = { ...item, position: newPos }
    avoidingIds.add(itemId)
  }
}
```

**作用**：拖拽元素离开后，立即让元素恢复原位

## 📊 效果对比

### 优化前
```
拖拽元素经过 A
  → A 避让 (移到新位置)
拖拽元素离开 A
  → A 停留在新位置 ❌ (粘住了)
```

### 优化后
```
拖拽元素经过 A
  → A 避让 (移到新位置) ✅
拖拽元素离开 A
  → A 立即恢复原位 ✅ (丝滑！)
```

## 🎯 评判标准

### ✅ 丝滑度合格标准
1. **独立性**：每个元素独立运动，不相互影响
2. **及时性**：拖拽元素离开后，0.25s 内恢复原位
3. **流畅性**：所有动画流畅，没有卡顿或跳跃
4. **视觉连贯**：过渡动画自然，符合物理直觉

### ❌ 需要继续优化的情况
1. 元素仍然"粘"在避让位置
2. 恢复动画有明显延迟或卡顿
3. 快速拖拽时元素跟不上
4. 视觉反馈不清晰或混乱
