## 🎯 功能概述

**模拟模式**是一个专门为测试和演示设计的功能，允许你在**不需要OpenAI API密钥**的情况下体验"文章转工作流"功能。

### 特点

✅ **无需API密钥** - 完全离线运行，不调用任何外部AI服务
✅ **快速响应** - 1-2秒即可生成工作流（真实模式需要10-30秒）
✅ **预设模板** - 根据URL关键词返回精心设计的工作流模板
✅ **完全免费** - 不消耗任何积分或API费用
✅ **适合测试** - 用于开发、演示、教学等场景

## 🔍 URL关键词匹配规则

模拟服务会根据URL中的关键词自动选择合适的工作流模板：

| URL包含关键词 | 生成的工作流类型 |
|-------------|---------------|
| `blog`, `write`, `写作`, `博客` | 博客写作工作流 |
| `data`, `analysis`, `数据`, `分析` | 数据分析工作流 |
| `design`, `product`, `设计`, `产品` | 产品设计工作流 |
| 其他任意URL | 通用工作流 |

### 示例

```bash
# 这些URL都会生成博客写作工作流
https://blog.example.com/article
https://example.com/how-to-write
```

## ❓ 常见问题

### Q1: 模拟模式生成的工作流可以执行吗？

**A:** 可以！生成的工作流结构完全相同，只是内容是预设的模板。你可以：
- ✅ 在编辑器中查看和编辑
- ✅ 保存到数据库
- ✅ 执行工作流（如果配置了节点参数）
- ✅ 导出和分享

### Q2: 模拟模式会访问网络吗？

**A:** 不会。模拟模式完全在本地运行，不会：
- ❌ 抓取真实的文章内容
- ❌ 调用OpenAI API
- ❌ 发送任何外部请求

只会根据URL关键词返回预设的模板数据。

### Q3: 可以自定义模拟模板吗？

**A:** 可以！编辑以下文件：

```
backend/src/services/mockArticleAnalysisService.ts
```

在 `analyzeArticleAndExtractWorkflow` 方法中添加新的 URL 匹配规则和模板。

### Q4: 模拟模式的工作流标题为什么带 [测试] 前缀？

**A:** 这是为了区分模拟生成和真实生成的工作流。你可以在工作流列表中快速识别哪些是测试数据。

如果不需要这个前缀，可以在后端代码中删除：

```typescript
// backend/src/routes/workflows.ts 第467行
title: `[测试] ${analysisResult.workflowTitle}`,  // 删除 [测试]
```

### Q5: 如何切换回真实模式？

**A:** 在前端页面：
1. 取消勾选 "🧪 测试模式已启用"
2. 确保后端配置了 `OPENAI_API_KEY`
3. 输入真实的文章URL
4. 点击生成
