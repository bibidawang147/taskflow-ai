# 探索页 UI/UX 优化说明

## 视觉层次
- 构建 `explore-hero` 顶部英雄区，利用渐变背景与玻璃拟态标签强化入口识别，并集中说明价值主张。
- 统一英雄、排行榜、卡片圆角为 16/12/10/6px，并使用双层阴影（0 1px 3px / 0 8px 24px）提升质感。
- 排行榜改为三列并排展示，减少空白；每列独立标题、描述与“查看更多”操作。
- 新增 token（字体层级、间距、渐变、按钮、标签）驱动页面视觉，使模块间厚度、密度保持一致。
- 页面背景加入轻量径向渐变与 Hero 层级的发光装饰，整体观感更具品牌记忆点。

## 交互体验
- 排行榜改为并列卡片，交互由按钮触发导航； hover 进入时带轻微缩进与灰底提示。
- 工作包、热门主题卡均采用 24px padding + 12px gap，hover 上浮 2-4px 并加强阴影。
- CTA、查看更多链接、主题芯片统一 44px 高度/对齐栈，hover 与 focus-visible 均有视觉反馈。

## 信息架构
- 归一排行榜数据结构，在 `rankingTabs` 中定义标签配置与 Metrics 元信息，便于后续扩展新榜单类型。
- 将热门主题卡片拆分为主题摘要 + 子类芯片两段，通过 `theme-card__subgrid` 自适应布局，突出数量与层级关系。
- 工作包卡片的核心指标与标签分区展示，避免信息挤压，并保留原有“查看详情”“一键导入”路径。
- 三大板块垂直间距统一为 48px，卡片内 padding 20-24px，保持高密度同时避免压迫感。

## 响应式与暗色模式
- 新增 1024 / 768 / 560px 多段断点，栅格在窄屏下自动切换为单列；排行榜行改为两行布局以保持可读性。
- 使用 `prefers-color-scheme: dark` 统一暗色背景与文字对比度，配合 `color-mix` 生成低亮度配色，保证夜间体验。
- 通过 `prefers-reduced-motion` 兼顾低动效偏好，将主交互动效降至最小。

## 可访问性与细节
- 引入 `.sr-only` 用于趋势描述的屏幕阅读器文本；列表/标签区均使用语义化元素（`ol`、`article`、`role="list"` 等）。
- 所有交互元素提供 `focus-visible` 与键盘触发逻辑（如主题卡 Enter/Space 导航）。
- 统一按钮的主/次样式，新增 “HOT” 标记的胶囊态视觉强化区分度。
- 排行榜趋势改为胶囊徽章，箭头使用品牌色；标签 hover 带缩放反馈提升可点击性。
- 排行榜的趋势状态采用胶囊徽章与色彩提示，配合 tabular 数字格式凸显数据差异。

## 技术实现摘要
- 重写 `ExplorePage` 排行榜结构，移除 Tab 逻辑并输出三列配置；数据仍通过 memo 缓存格式化器处理。
- `PopularWorkPackageCard` 换成纯 CSS 卡片结构（24px padding + stats-row + tag 栈），按钮复用 token。
- 新建的 `explore.css` 在 `:root` 管理 token，按模块划分注释，确保响应式 1200/768 阶段切换。

## 后续建议
1. 接入真实的排行榜数据接口，结合切换标签支持按时间维度筛选。
2. 为工作包/主题增加骨架屏与空状态组件，完善数据加载体验。
3. 引入可视化埋点，分析热门区块点击与曝光，持续迭代信息密度。
