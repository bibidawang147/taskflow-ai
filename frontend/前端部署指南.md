# 前端部署到阿里云完整指南

## 📋 部署概览

**目标**：把前端部署到阿里云服务器 47.93.218.80，让用户能通过浏览器访问

**架构**：
```
用户浏览器
    ↓
阿里云服务器 (47.93.218.80)
    ↓
Nginx (前端静态文件 + 反向代理后端API)
    ↓
后端 Node.js (localhost:3000)
```

**预计时间**：2-3 小时

## 第一步：检查前端配置 (本地)

### 1.1 检查 API 地址配置

**在本地执行：**

```bash
cd /Users/star/Desktop/开发文件夹/workflow-platform/frontend
```

**查找 API 配置文件：**

```bash
# 查看是否有环境变量配置
cat .env 2>/dev/null || echo "没有 .env 文件"
cat .env.production 2>/dev/null || echo "没有 .env.production 文件"

# 查看 API 配置
grep -r "localhost:3000" src/ || grep -r "http://localhost" src/
```

### 1.2 创建生产环境配置

**创建 `.env.production` 文件：**

```bash
cat > .env.production << 'EOF'
# 生产环境配置
VITE_API_BASE_URL=http://47.93.218.80:3000
VITE_APP_ENV=production
EOF
```

**如果你有域名（例如 workflow.yourdomain.com）：**

```bash
cat > .env.production << 'EOF'
VITE_API_BASE_URL=https://workflow.yourdomain.com/api
VITE_APP_ENV=production
EOF
```

### 1.3 更新 API 配置代码

**检查 `src/services/api.ts`：**

需要确保使用环境变量：

```typescript
// src/services/api.ts
const baseURL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:3000';

const api = axios.create({
  baseURL: baseURL,
  timeout: 30000,
});
```

## 第四步：服务器配置 (SSH 登录)

### 4.1 SSH 登录服务器

```bash
ssh -i /Users/star/Desktop/工作流网站.pem root@47.93.218.80
```

### 4.2 安装 Nginx

```bash
# 安装
yum install -y nginx

# 启动
systemctl start nginx
systemctl enable nginx

# 验证
nginx -v
```

### 4.3 解压前端文件

**如果上传的是压缩包：**

```bash
mkdir -p /var/www/workflow-platform
cd /var/www/workflow-platform
tar -xzf /root/frontend-dist.tar.gz
```

**如果上传的是目录：**

```bash
mv /root/workflow-frontend /var/www/workflow-platform
```

### 4.4 配置 Nginx

**创建配置文件：**

```bash
cat > /etc/nginx/conf.d/workflow.conf << 'EOF'
server {
    listen 80;
    server_name 47.93.218.80;  # 或者你的域名

    # 前端静态文件
    root /var/www/workflow-platform;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;

    # 前端路由（SPA）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }

    # 静态文件缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

### 4.5 测试并重启 Nginx

```bash
# 测试配置
nginx -t

# 重启
systemctl restart nginx

# 查看状态
systemctl status nginx
```

## 第五步：配置防火墙和安全组

### 5.1 服务器防火墙

```bash
# 开放 80 端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --reload

# 或者用 iptables
iptables -I INPUT -p tcp --dport 80 -j ACCEPT
```

### 5.2 阿里云安全组

**登录阿里云控制台：**
1. 访问 https://ecs.console.aliyun.com
2. 找到 47.93.218.80 实例
3. 安全组 → 配置规则
4. 添加入站规则：
   - 端口：80/80
   - 授权对象：0.0.0.0/0
   - 描述：HTTP

## 🔧 故障排除

### 问题1：404 Not Found

**原因**：Nginx 配置错误或文件路径错误

**解决**：

```bash
# 检查文件
ls -la /var/www/workflow-platform/

# 检查 Nginx 日志
tail -f /var/log/nginx/error.log

# 检查配置
nginx -t
```

### 问题2：API 调用失败 (CORS)

**原因**：后端 CORS 未配置

**解决**：

编辑后端 `.env`，添加：

```bash
ALLOWED_ORIGINS=http://47.93.218.80
```

重启后端。

### 问题3：前端白屏

**原因**：API 地址配置错误

**解决**：

检查浏览器控制台，看 API 请求地址。

重新构建前端：

```bash
# 本地
VITE_API_BASE_URL=http://47.93.218.80:3000 npm run build
# 重新上传
```

### 问题4：Nginx 无法启动

**原因**：端口被占用

**解决**：

```bash
# 查看端口
lsof -i :80

# 停止占用的进程
systemctl stop httpd  # 如果是 Apache
```
