# 使用密钥部署命令

> 服务器IP：**47.93.218.80**
> 密钥文件：**工作流网站.pem**（或 **工作流网站**）
>
> 以下所有命令都可以直接复制粘贴使用！

---

## 准备工作：设置密钥权限

### 1. 进入项目目录

```bash
cd /Users/star/Desktop/开发文件夹/workflow-platform/backend
```

### 2. 设置密钥文件权限（必须执行！）

如果文件名有 .pem 扩展名：
```bash
chmod 400 ~/Downloads/工作流网站.pem
```

如果文件名没有扩展名：
```bash
chmod 400 ~/Downloads/工作流网站
```

---

## 第一步：上传脚本到服务器

### 如果文件名是"工作流网站.pem"：

```bash
scp -i ~/Downloads/工作流网站.pem aliyun-setup.sh root@47.93.218.80:/root/
```

### 如果文件名是"工作流网站"（没有.pem）：

```bash
scp -i ~/Downloads/工作流网站 aliyun-setup.sh root@47.93.218.80:/root/
```

**不会提示输入密码，直接上传！**

成功会显示：
```
aliyun-setup.sh                100%   5KB   1.2MB/s   00:00
```

---

## 第二步：登录到服务器

### 如果文件名是"工作流网站.pem"：

```bash
ssh -i ~/Downloads/工作流网站.pem root@47.93.218.80
```

### 如果文件名是"工作流网站"（没有.pem）：

```bash
ssh -i ~/Downloads/工作流网站 root@47.93.218.80
```

**不会提示输入密码，直接登录！**

登录成功后，命令提示符会变成：
```
[root@iZ... ~]#
```

---

## 第三步：运行部署脚本（在服务器上）

登录服务器后，依次执行：

```bash
cd /root
```

```bash
chmod +x aliyun-setup.sh
```

```bash
./aliyun-setup.sh
```

等待2-5分钟，看到 "🎉 部署完成！" 就成功了！

---

## 第四步：退出服务器

```bash
exit
```

---

## 第五步：初始化数据库（在你的电脑上）

确保在项目目录：
```bash
cd /Users/star/Desktop/开发文件夹/workflow-platform/backend
```

生成 Prisma Client：
```bash
npm run db:generate
```

初始化数据库表：
```bash
npm run db:push
```

---

## 第六步：启动和测试

启动后端：
```bash
npm run dev
```

在新终端测试：
```bash
curl http://localhost:3000/health
```

看到 `{"status":"healthy"}` 就成功了！

---

## 常用管理命令（使用密钥）

### 查看数据库状态：

如果文件名是"工作流网站.pem"：
```bash
ssh -i ~/Downloads/工作流网站.pem root@47.93.218.80 "cd /root/workflow-db && docker-compose ps"
```

如果文件名是"工作流网站"：
```bash
ssh -i ~/Downloads/工作流网站 root@47.93.218.80 "cd /root/workflow-db && docker-compose ps"
```

### 查看日志：

```bash
ssh -i ~/Downloads/工作流网站.pem root@47.93.218.80
cd /root/workflow-db
docker-compose logs -f
```

### 重启数据库：

```bash
ssh -i ~/Downloads/工作流网站.pem root@47.93.218.80
cd /root/workflow-db
docker-compose restart
```

## 如果遇到"权限被拒绝"错误

执行：
```bash
chmod 400 ~/Downloads/工作流网站.pem
```

或
```bash
chmod 400 ~/Downloads/工作流网站
```

---

## 可选：把密钥文件移到更安全的位置

```bash
mkdir -p ~/.ssh
mv ~/Downloads/工作流网站.pem ~/.ssh/
chmod 400 ~/.ssh/工作流网站.pem
```

之后使用：
```bash
ssh -i ~/.ssh/工作流网站.pem root@47.93.218.80
```
