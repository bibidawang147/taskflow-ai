# 阿里云部署步骤（5分钟完成）

## 📋 前提条件

- ✅ 你有一台阿里云 ECS 服务器
- ✅ 可以 SSH 登录（root 或 sudo 权限）
- ✅ 服务器操作系统：Ubuntu / CentOS / Debian

## 🚀 快速部署（3条命令）

### 步骤 1: 上传脚本到服务器

在**本地电脑**执行：

```bash
# 替换 your-server-ip 为你的服务器IP
scp aliyun-setup.sh root@your-server-ip:/root/
```

示例：
```bash
scp aliyun-setup.sh root@47.100.123.456:/root/
```

### 步骤 2: SSH 登录服务器

```bash
ssh root@your-server-ip
```

### 步骤 3: 运行部署脚本

在**服务器**上执行：

```bash
cd /root
chmod +x aliyun-setup.sh
./aliyun-setup.sh
```

**脚本会自动完成：**
- ✅ 安装 Docker 和 Docker Compose
- ✅ 创建工作目录 /root/workflow-db
- ✅ 创建 docker-compose.yml 配置
- ✅ 启动 PostgreSQL 和 Redis
- ✅ 检查服务状态

等待 2-3 分钟，看到 "🎉 部署完成！" 即表示成功。

## 📝 步骤 4: 初始化数据库

回到**本地电脑**项目目录，执行：

```bash

cat .env | grep DATABASE_URL
```

看到 "Your database is now in sync with your schema" 即表示成功。

## 🔒 安全配置（可选但推荐）

### 1. 修改数据库密码

在服务器上：

```bash
nano /root/workflow-db/docker-compose.yml

# 修改这一行：
# POSTGRES_PASSWORD: 你的新密码

# 重启服务
cd /root/workflow-db
docker-compose restart postgres
```

然后更新本地 `.env` 文件中的密码。

### 2. 配置阿里云安全组

登录阿里云控制台 → ECS → 安全组 → 配置规则

**入站规则：**
- ✅ 22/tcp - SSH（限制来源IP更安全）
- ✅ 3000/tcp - Backend API（如果后端也在同一服务器）
- ❌ **不要开放** 5432（PostgreSQL）到公网
- ❌ **不要开放** 6379（Redis）到公网

### 3. 配置服务器防火墙

```bash
# Ubuntu UFW
ufw allow 22/tcp
ufw allow 3000/tcp
ufw enable

# CentOS firewalld
firewall-cmd --permanent --add-port=22/tcp
firewall-cmd --permanent --add-port=3000/tcp
firewall-cmd --reload
```

## 💾 自动备份设置（可选）

在服务器上设置每天自动备份：

```bash
# 编辑 crontab
crontab -e

# 添加以下内容（每天凌晨 2 点备份）
0 2 * * * cd /root/workflow-db && docker exec workflow-postgres pg_dump -U workflow_user workflow_platform | gzip > backups/backup_$(date +\%Y\%m\%d).sql.gz

# 清理 30 天前的备份
0 3 * * * find /root/workflow-db/backups -name "backup_*.sql.gz" -mtime +30 -delete
```

## 🆘 故障排除

### 问题 1: 无法连接数据库

```bash
# 检查容器状态
cd /root/workflow-db
docker-compose ps

# 查看 PostgreSQL 日志
docker-compose logs postgres

# 测试连接
docker exec workflow-postgres pg_isready -U workflow_user

# 检查端口
netstat -tlnp | grep 5432
```

### 问题 2: 端口被占用

```bash
# 查看占用端口的进程
lsof -i :5432
lsof -i :6379

# 停止占用端口的进程（小心！）
kill <PID>
```

### 问题 3: 磁盘空间不足

```bash
# 检查磁盘空间
df -h

# 清理 Docker
docker system prune -a --volumes

# 清理旧备份
rm -f /root/workflow-db/backups/backup_*.sql.gz
```

### 问题 4: 内存不足

```bash
# 检查内存使用
docker stats

# 降低 PostgreSQL 内存（编辑 docker-compose.yml）
# 将 shared_buffers 从 256MB 改为 128MB
# 将 max_connections 从 200 改为 100
```
