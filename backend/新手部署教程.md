# 新手部署教程 - 零基础版

> 📝 **说明**：这个教程假设你完全没有服务器操作经验，会手把手教你每一步。

---

## 📋 准备工作

### 你需要知道的信息

1. **你的阿里云服务器IP地址**
   - 登录阿里云控制台：https://ecs.console.aliyun.com
   - 找到你的 ECS 实例
   - 复制 "公网IP" （例如：47.100.123.456）

2. **你的服务器登录密码**
   - 如果忘记了密码，可以在阿里云控制台重置

## 第一步：准备部署脚本（在你的电脑上）

### 1. 打开终端

**Mac 用户：**
- 按 `Command + 空格` 打开 Spotlight
- 输入 `terminal` 回车

**Windows 用户：**
- 按 `Win + R`
- 输入 `cmd` 回车

### 2. 进入项目目录

在终端中输入（直接复制粘贴）：

```bash
cd /Users/star/Desktop/开发文件夹/workflow-platform/backend
```

按回车。

### 3. 检查文件是否存在

输入：

```bash
ls -la aliyun-setup.sh
```

应该看到类似这样的输出：
```
-rwxr-xr-x  1 star  staff  5234 Nov  3 10:30 aliyun-setup.sh
```

如果看到 "No such file or directory"，告诉我，我会帮你解决。

---

## 第二步：上传脚本到服务器

### 重要提示
⚠️ 接下来的命令中，你需要把 `YOUR_SERVER_IP` 替换成你的真实服务器IP地址

### 上传命令

**完整命令格式：**
```bash
scp aliyun-setup.sh root@YOUR_SERVER_IP:/root/
```

**示例（假设你的服务器IP是 47.100.123.456）：**
```bash
scp aliyun-setup.sh root@47.100.123.456:/root/
```

### 执行步骤：

1. **替换IP地址**
   - 复制上面的命令
   - 把 `YOUR_SERVER_IP` 改成你的真实IP
   - 粘贴到终端

2. **输入密码**
   - 按回车后，会提示输入密码
   - 输入你的服务器root密码（**注意：输入时不会显示任何字符，这是正常的**）
   - 输入完按回车

3. **等待上传**
   - 应该很快完成（几秒钟）
   - 看到类似这样的输出表示成功：
   ```
   aliyun-setup.sh                100%   5KB   1.2MB/s   00:00
   ```

### 可能遇到的问题

**问题1: "Permission denied"**
- 原因：密码输入错误
- 解决：重新输入，注意密码区分大小写

**问题2: "Connection refused" 或 "Connection timed out"**
- 原因：服务器IP错误，或者防火墙没开放22端口
- 解决：检查IP地址是否正确，在阿里云控制台检查安全组是否开放了22端口

**问题3: "Host key verification failed"**
- 原因：第一次连接这台服务器
- 解决：输入 `yes` 后回车，然后重新执行上传命令

## 第三步：登录到服务器

### 登录命令

**完整命令格式：**
```bash
ssh root@YOUR_SERVER_IP
```

**示例（假设你的服务器IP是 47.100.123.456）：**
```bash
ssh root@47.100.123.456
```

### 执行步骤：

1. **输入命令**
   - 替换成你的真实IP
   - 粘贴到终端
   - 按回车

2. **输入密码**
   - 输入服务器root密码（不会显示字符）
   - 按回车

3. **登录成功的标志**
   - 命令提示符变成类似这样：
   ```
   [root@iZ2ze... ~]#
   ```
   或
   ```
   root@ubuntu:~#
   ```

   这表示你已经在服务器上了！

### 可能遇到的问题

**问题: 总是提示密码错误**
- 在阿里云控制台重置root密码
- 等待1-2分钟后重试

## 第四步：运行部署脚本（在服务器上）

⚠️ **注意**：现在你的命令是在服务器上执行的，不是在你的电脑上！

### 1. 进入root目录

输入：
```bash

```

### 2. 检查脚本是否上传成功

输入：

```

```

### 3. 给脚本执行权限

输入：
```bash
chmod +x aliyun-setup.sh
```

输入：

### 可能遇到的问题

**问题1: "Docker installation failed"**
- 原因：网络问题或系统不支持
- 解决：重新运行脚本 `./aliyun-setup.sh`

**问题2: "port is already allocated"**
- 原因：端口被占用
- 解决：
  ```bash
  # 查看占用端口的进程
  netstat -tlnp | grep 5432

  # 如果有其他PostgreSQL在运行，停止它
  systemctl stop postgresql

  # 重新运行脚本
  ./aliyun-setup.sh
  ```

**问题3: 脚本执行到一半卡住**
- 按 `Ctrl + C` 终止
- 重新运行 `./aliyun-setup.sh`

## 第六步：初始化数据库（在你的电脑上）

⚠️ **注意**：现在你的命令是在你的电脑上执行的，不是在服务器上！

### 1. 确保在项目目录

输入：
```bash
cd /Users/star/Desktop/开发文件夹/workflow-platform/backend
```
按回车。

### 2. 检查 .env 配置

输入：
```bash
cat .env | grep DATABASE_URL
```
按回车。

应该看到：
```
DATABASE_URL="postgresql://workflow_user:WorkflowSecure2024!@#@localhost:5432/workflow_platform?schema=public"
```

⚠️ **重要**：这里的 `localhost` 在服务器上才有效。如果你的后端在本地开发，需要改成服务器IP。

**如果后端在本地开发（而不是部署在服务器上）：**

你需要修改 `.env` 文件：

输入：
```bash
nano .env
```

找到这一行：
```
DATABASE_URL="postgresql://workflow_user:WorkflowSecure2024!@#@localhost:5432/workflow_platform?schema=public"
```

把 `localhost` 改成你的服务器IP：
```
DATABASE_URL="postgresql://workflow_user:WorkflowSecure2024!@#@YOUR_SERVER_IP:5432/workflow_platform?schema=public"
```

按 `Ctrl + O` 保存，按回车确认，按 `Ctrl + X` 退出。

### 3. 生成 Prisma Client

输入：
```bash
npm run db:generate
```
按回车。

应该看到：
```
✔ Generated Prisma Client
```

### 4. 创建数据库表

输入：
```bash
npm run db:push
```
按回车。

应该看到：
```
🚀 Your database is now in sync with your Prisma schema.
```

**如果看到错误：**

```
Error: Can't reach database server at `localhost:5432`
```

这表示：
1. 如果后端在本地：你需要开放服务器5432端口（见下面的"开放端口"部分）
2. 如果后端在服务器：忽略这个错误，在服务器上运行这些命令

## 🔒 附加：开放数据库端口（仅当后端在本地时需要）

### 重要安全说明

⚠️ 开放数据库端口会有安全风险！只在以下情况才需要：
- 后端在本地开发，数据库在服务器上
- 临时开发测试使用

**生产环境不应该开放数据库端口！**

### 方法1：阿里云安全组（推荐）

1. 登录阿里云控制台：https://ecs.console.aliyun.com
2. 点击你的ECS实例
3. 点击"安全组"标签
4. 点击"配置规则"
5. 点击"添加安全组规则"
6. 填写：
   - 端口范围：`5432/5432`
   - 授权对象：`你的本地IP/32` （更安全）或 `0.0.0.0/0` （允许所有，不安全）
   - 描述：`PostgreSQL开发测试`
7. 点击"确定"

### 方法2：服务器防火墙

SSH登录服务器，输入：

**Ubuntu/Debian:**
```bash
ufw allow 5432/tcp
```

**CentOS:**
```bash
firewall-cmd --permanent --add-port=5432/tcp
firewall-cmd --reload
```

## 🆘 遇到问题怎么办？

### 查看服务器日志

SSH登录服务器，输入：
```bash
cd /root/workflow-db
docker-compose logs -f
```

按 `Ctrl + C` 退出日志查看。

### 重启服务

```bash
cd /root/workflow-db
docker-compose restart
```

### 完全重新部署

```bash
cd /root/workflow-db
docker-compose down -v
cd /root
./aliyun-setup.sh
```
