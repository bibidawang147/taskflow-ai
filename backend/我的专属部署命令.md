# 我的专属部署命令

> 服务器IP：**47.93.218.80**
>
> 以下所有命令都可以直接复制粘贴使用！

---

## 第一步：上传脚本到服务器

### 在你的电脑终端执行：

```bash
cd /Users/star/Desktop/开发文件夹/workflow-platform/backend
```

然后执行：

```bash
scp aliyun-setup.sh root@47.93.218.80:/root/
```

**会提示输入密码**：输入你的服务器root密码（输入时不会显示，这是正常的）

---

## 第二步：登录到服务器

```bash
ssh root@47.93.218.80
```

**会提示输入密码**：输入你的服务器root密码

---

## 第三步：运行部署脚本（在服务器上执行）

登录服务器后，依次执行：

```bash
cd /root
```

```bash
chmod +x aliyun-setup.sh
```

```bash
./aliyun-setup.sh
```

等待2-5分钟，看到 "🎉 部署完成！" 就成功了！

---

## 第四步：退出服务器

```bash
exit
```

---

## 第五步：配置数据库连接（在你的电脑上执行）

### 重要：选择你的部署方式

**方式A：后端也部署在服务器上**
- 保持 .env 中的 `localhost` 不变
- DATABASE_URL 使用 `localhost:5432`

**方式B：后端在本地开发，数据库在服务器上（临时开发用）**
- 需要把 `localhost` 改成 `47.93.218.80`
- 需要开放服务器5432端口

---

### 如果是方式B（本地开发），修改 .env：

```bash
nano .env
```

找到这一行：
```
DATABASE_URL="postgresql://workflow_user:WorkflowSecure2024!@#@localhost:5432/workflow_platform?schema=public"
```

改成：
```
DATABASE_URL="postgresql://workflow_user:WorkflowSecure2024!@#@47.93.218.80:5432/workflow_platform?schema=public"
```

按 `Ctrl + O` 保存，按回车，按 `Ctrl + X` 退出

---

## 第六步：初始化数据库（在你的电脑上执行）

```bash
npm run db:generate
```

```bash
npm run db:push
```

---

## 第七步：测试

```bash
npm run dev
```

在新终端测试：

```bash
curl http://localhost:3000/health
```

看到 `{"status":"healthy"}` 就成功了！

## 常用管理命令

### 查看服务器上的数据库状态：

```bash

cd /root/workflow-db
docker-compose ps
```

### 查看日志：

```bash
ssh root@47.93.218.80
cd /root/workflow-db
docker-compose logs -f
```

### 重启数据库：

```bash
ssh root@47.93.218.80
cd /root/workflow-db
docker-compose restart
```

## 如果需要开放5432端口（仅方式B需要）

登录阿里云控制台：
1. 访问：https://ecs.console.aliyun.com
2. 点击你的实例（IP: 47.93.218.80）
3. 点击"安全组" → "配置规则"
4. 添加规则：
   - 端口：5432/5432
   - 授权对象：0.0.0.0/0
   - 描述：PostgreSQL临时开发

⚠️ **安全提醒**：生产环境不要开放5432端口！
