# 域名配置指南

完整配置你的域名以访问工作流平台

## 准备工作

确保你有：
- ✅ 已购买的域名（例如：`yourdomain.com`）
- ✅ 域名管理面板的访问权限
- ✅ 服务器 IP：`47.93.218.80`
- ✅ SSH 访问权限

## 步骤 1: 配置域名 DNS 解析

### 1.1 登录域名服务商管理面板

常见域名服务商：
- 阿里云（万网）：https://dns.console.aliyun.com
- 腾讯云：https://console.cloud.tencent.com/cns
- Cloudflare：https://dash.cloudflare.com
- GoDaddy：https://dcc.godaddy.com/manage/dns

### 1.2 添加 DNS 记录

添加以下记录：

#### 主域名记录
```
类型: A
主机记录: @
记录值: 47.93.218.80
TTL: 600（或默认）
```

#### www 子域名记录（可选，推荐）
```
类型: A
主机记录: www
记录值: 47.93.218.80
TTL: 600（或默认）
```

#### API 子域名记录（可选，如果前后端完全分离）
```
类型: A
主机记录: api
记录值: 47.93.218.80
TTL: 600（或默认）
```

### 1.3 验证 DNS 解析

等待 5-10 分钟后，在本地终端测试：

```bash
# 测试主域名
ping yourdomain.com

# 测试 www 子域名
ping www.yourdomain.com

# 或使用 nslookup
nslookup yourdomain.com
```

✅ 如果返回 `47.93.218.80`，说明 DNS 解析成功！

## 步骤 2: 配置服务器 Nginx

### 2.1 连接到服务器

```bash
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80
```

### 2.2 创建 Nginx 配置文件

```bash
# 创建配置文件（替换 yourdomain.com 为你的实际域名）
nano /etc/nginx/sites-available/workflow-platform
```

### 2.3 配置内容（HTTP，先不配置 HTTPS）

复制以下内容，**替换 `yourdomain.com` 为你的实际域名**：

```nginx
# 后端 API 服务
upstream backend {
    server 127.0.0.1:3001;
    keepalive 64;
}

# HTTP 服务器配置
server {
    listen 80;
    listen [::]:80;
    server_name yourdomain.com www.yourdomain.com;

    # 日志
    access_log /var/log/nginx/workflow-platform-access.log;
    error_log /var/log/nginx/workflow-platform-error.log;

    # 前端静态文件
    location / {
        root /var/www/workflow-platform;
        index index.html;
        try_files $uri $uri/ /index.html;

        # 添加缓存头
        add_header Cache-Control "public, max-age=31536000" always;
    }

    # 后端 API 代理
    location /api {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 文件上传接口（如果有）
    location /uploads {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # 增加上传大小限制
        client_max_body_size 50M;
    }

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        root /var/www/workflow-platform;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

### 2.4 启用配置并测试

```bash
# 创建软链接启用配置
ln -s /etc/nginx/sites-available/workflow-platform /etc/nginx/sites-enabled/

# 测试配置是否正确
nginx -t

# 如果测试通过，重启 Nginx
systemctl restart nginx

# 查看 Nginx 状态
systemctl status nginx
```

### 2.5 测试 HTTP 访问

在浏览器访问：`http://yourdomain.com`

✅ 如果能看到网站，说明配置成功！

## 步骤 3: 配置 SSL 证书（HTTPS）

### 3.1 安装 Certbot

```bash
# 安装 Certbot（Let's Encrypt 工具）
apt update
apt install certbot python3-certbot-nginx -y
```

### 3.2 自动获取并配置 SSL 证书

```bash
# 替换 yourdomain.com 为你的实际域名
certbot --nginx -d yourdomain.com -d www.yourdomain.com

# 按提示操作：
# 1. 输入邮箱（用于证书过期提醒）
# 2. 同意服务条款：输入 Y
# 3. 是否分享邮箱：输入 N
# 4. 选择 2（重定向 HTTP 到 HTTPS）
```

### 3.3 测试自动续期

```bash
# 测试证书自动续期（不会真的续期）
certbot renew --dry-run
```

✅ 如果测试成功，证书会在过期前自动续期！

### 3.4 验证 HTTPS 访问

在浏览器访问：`https://yourdomain.com`

- ✅ 地址栏显示锁图标
- ✅ 网站正常访问
- ✅ HTTP 自动跳转到 HTTPS

```typescript
  timeout: 30000,
  headers: {
    'Content-Type': 'application/json',
  },
});

export default api;
```

```typescript
    'https://yourdomain.com',
    'https://www.yourdomain.com',

  ].filter(Boolean),
  credentials: true,
}));
```

## 常见问题排查

### Q1: 访问域名显示 "无法访问此网站"

**可能原因：**
1. DNS 解析未生效（等待 10-30 分钟）
2. 防火墙阻止了 80/443 端口
3. Nginx 未启动

**解决方法：**
```bash
# 测试 DNS
ping yourdomain.com

# 检查 Nginx
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80 "systemctl status nginx"

# 检查防火墙
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80 "ufw status"
```

### Q2: HTTP 可以访问，HTTPS 不行

**可能原因：**
1. SSL 证书获取失败
2. 防火墙未开放 443 端口

**解决方法：**
```bash
# 重新获取证书
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80
certbot --nginx -d yourdomain.com -d www.yourdomain.com --force-renewal

# 开放 443 端口
ufw allow 443/tcp
```

### Q3: 网站能访问但 API 调用失败

**可能原因：**
1. CORS 配置不正确
2. 后端服务未运行
3. 前端 API 地址配置错误

**解决方法：**
```bash
# 检查后端服务
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80 "pm2 list"

# 查看后端日志
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80 "pm2 logs workflow-backend --lines 50"

# 检查浏览器控制台错误
# 打开浏览器 F12 -> Console 查看具体错误
```

### Q4: SSL 证书获取失败

**可能原因：**
1. 域名解析未指向正确的服务器
2. 80 端口未开放
3. 已有其他 Web 服务占用 80 端口

**解决方法：**
```bash
# 确保 80 端口可访问
curl -I http://yourdomain.com

# 检查端口占用
ssh -i ~/Desktop/工作流*.pem root@47.93.218.80 "netstat -tlnp | grep :80"

# 手动模式获取证书
certbot certonly --standalone -d yourdomain.com -d www.yourdomain.com
```

## 性能优化建议

### 1. 启用 Gzip 压缩

在 Nginx 配置中添加：

```nginx
gzip on;
gzip_vary on;
gzip_min_length 1024;
gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml+rss application/json application/javascript;
```

### 2. 配置 CDN（可选）

如果用户遍布全国/全球，可以使用 CDN 加速：
- 阿里云 CDN
- 腾讯云 CDN
- Cloudflare CDN（免费）

```bash
  cd /root/workflow-backend
```
